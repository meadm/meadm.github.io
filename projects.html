What I typically need to start

A quick kickoff saves time. If you don’t have everything, that’s fine — we can iterate.

Inputs

  • Data location (FASTQs / BAMs / matrices / VCFs)
  • Metadata + sample sheet
  • Reference build + annotations (if relevant)
  • Any existing pipeline or preferred tools

Constraints

  • Environment (HPC, cloud, local) + access constraints
  • Timelines and “definition of done”
  • Confidentiality and sharing expectations
  • Preferred deliverable format (report, repo, both)
